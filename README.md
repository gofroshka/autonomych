# Автономыч

Десктопное приложение, в которое ты описываешь идею проекта — а оно само его пишет. Без «approve», без «continue», без сидения над терминалом. Открыл, кинул промпт, ушёл пить кофе. Вернулся — нажал «Покажи, что есть».

Под капотом — рой агентов поверх `claude` CLI. Product Owner придумывает фичи, Architect режет на задачи с зависимостями, Specialists пишут код параллельно в своих git worktree, Reviewer мерджит обратно в main. И так по кругу, итерация за итерацией, пока ты не остановишь.

В любой момент жмёшь **«Покажи, что есть»** — текущая итерация добегает до конца, поднимается dev-сервер сгенерированного проекта прямо в приложении, ты тыкаешь руками. Не нравится — пишешь в чат «убери вот это, добавь вон то», жмёшь **«Продолжаем»**, и Автономыч уходит в следующий виток уже с твоими правками.

## Зачем это вообще

Обычный vibe-coding выглядит как: ты пишешь промпт, ждёшь, читаешь diff, говоришь «ок», пишешь следующий промпт. И так часами. Половина времени уходит не на код, а на то, чтобы держать агента за руку.

Автономыч пробует другой режим: ты не дирижируешь каждым шагом, ты задаёшь направление и периодически смотришь, что получилось. Остальное он разруливает сам — декомпозицию, очерёдность, мерджи, фиксы после ревью, перезапуск зависших задач после краша.

Подходит, когда хочется:
- быстро получить рабочий MVP, не сидя над процессом
- посмотреть, до какого предела дотащит автономный агент, прежде чем понадобится твоё вмешательство
- сгенерить пет-проект, пока занимаешься чем-то другим

## Что нужно для запуска

- Rust toolchain (`rustup`)
- Node 18+
- `claude` CLI в PATH, авторизованный (`claude login` или `ANTHROPIC_API_KEY`)
- `git`

Дальше:

```bash
git clone https://github.com/gofroshka/autonomych.git
cd autonomych
npm install
npm run tauri:dev
```

На macOS соберётся `.app`/`.dmg` через `npm run tauri:build`. Linux/Windows технически должно работать, но я гонял только на маке — если попробуешь, скажи как.

## Как пользоваться

1. Создай проект, укажи папку, где будет жить код.
2. Опиши, что хочешь сделать — одной строкой или абзацем, как в обычном чате с агентом.
3. Жми Start. Дальше можно вообще закрыть окно (приложение продолжит крутиться) или смотреть в реальном времени: что делает каждый агент, граф задач, лог действий.
4. Когда захочешь посмотреть результат — кнопка «Покажи, что есть». Дождись, пока поднимется preview.
5. Тыкаешь приложение, пишешь курс-коррекцию (мягкую или жёсткую — «обязательно сделай Х»), жмёшь «Продолжаем». Уходит на следующий виток.

Можно нажать «Покажи, что есть» сразу после создания проекта — если код ещё не пишется (или работа на ранней стадии), Автономыч покажет состояние «как есть» и ждёт твоего пинка. Это удобно для первой курс-коррекции, пока он ещё не наворотил всякого.

## Что внутри

- Нативное Tauri-приложение на Rust. Бандл ~10-15 МБ в релизе, RAM на старте ~80 МБ.
- К Claude хожу напрямую через `claude` CLI в режиме `stream-json` — без отдельных SDK и Node-сайдкаров.
- Каждая задача специалиста — отдельный `git worktree` на своей ветке. Параллельная работа без конфликтов, мерджи через `--no-ff`, перед каждой итерацией ставится тег `autonomych/pre-iter-N` чтобы можно было откатиться.
- Crash recovery из коробки: убил приложение посреди итерации — при старте оно подберёт состояние, прибьёт сиротские worktree и preview-процессы, доиграет незавершённое.
- Если специалист реально упёрся — он пишет вопрос в специальном формате, его перехватывает Blocker Reviewer; либо отвечает сам (если очевидно), либо прокидывает тебе в UI и ждёт.
- Сбоку — чат с Overseer-агентом, который видит всё состояние проекта и может объяснить, что вообще происходит, или подсказать формулировку для steering.

## Честные ограничения

- **Стабильность.** Это pet-проект, не продакшен. UI работает, цикл крутится, но e2e на больших проектах я не гонял — если найдёшь, где ломается, открой issue.
- **`stream-json` от `claude` CLI — не публичный стабильный API.** Если Anthropic поменяет формат, парсер придётся подкручивать (файл `src-tauri/src/agents/claude.rs`).
- **Стоит денег.** Агенты крутятся в фоне и жгут токены. Wrap-up по кнопке существует не просто так — оставленный на ночь Автономыч может уйти в неплохой счёт.
- **Permission mode по умолчанию — `bypassPermissions`.** Это нужно, чтобы агенты могли запускать `Bash` без подтверждений. Запускай его на проектах, где не страшно — в отдельной папке, не в `$HOME`.
- **macOS only verified.** Preview-агент использует `setsid` для запуска dev-серверов — на Windows понадобится альтернатива.
- Стриминг чата с Overseer пока без инкрементального вывода — ждёт полного ответа. Поправлю.

# Автономыч

Десктопное приложение, в котором ты описываешь что строим — а оно само пишет проект. Без «approve», без «continue», без сидения над терминалом. Открыл, задал направление, ушёл пить кофе. Вернулся — нажал «Покажи, что есть».

Под капотом — рой агентов поверх `claude` CLI **или** `codex` CLI (на выбор, per-project). Product Owner придумывает фичи, Architect режет на задачи с зависимостями, Specialists пишут код параллельно в своих git worktree, Reviewer мерджит обратно в main и закрывает беклог. И так по кругу, итерация за итерацией, пока ты не остановишь.

В любой момент жмёшь **«Покажи, что есть»** — текущая итерация добегает до конца, поднимается dev-сервер сгенерированного проекта прямо в приложении, ты тыкаешь руками. Не нравится — кидаешь правки в беклог с категорией (баг / фича / идея / крит проблема), жмёшь **«Продолжаем»**. Автономыч сам решит что брать первым: критичные баги впереди фич, новый функционал — только когда система целая.

## Зачем это вообще

Обычный vibe-coding выглядит как: ты пишешь промпт, ждёшь, читаешь diff, говоришь «ок», пишешь следующий промпт. И так часами. Половина времени уходит не на код, а на то, чтобы держать агента за руку.

Автономыч пробует другой режим: ты не дирижируешь каждым шагом, ты задаёшь направление и периодически смотришь, что получилось. Остальное он разруливает сам — декомпозицию, очерёдность, мерджи, фиксы после ревью, перезапуск зависших задач после краша, ожидание восстановления лимитов провайдера.

Подходит, когда хочется:
- быстро получить рабочий MVP, не сидя над процессом
- посмотреть, до какого предела дотащит автономный агент, прежде чем понадобится твоё вмешательство
- сгенерить пет-проект, пока занимаешься чем-то другим

## Что нужно для запуска

- Rust toolchain (`rustup`)
- Node 18+
- `git`
- Один из двух CLI (или оба — выбирается на каждый проект):
  - `claude` CLI в PATH, авторизованный (`claude login` или `ANTHROPIC_API_KEY`)
  - `codex` CLI в PATH, авторизованный (`codex login`, ChatGPT-аккаунт или OpenAI API key)

Дальше:

```bash
git clone https://github.com/gofroshka/autonomych.git
cd autonomych
npm install
npm run tauri:dev
```

На macOS соберётся `.app`/`.dmg` через `npm run tauri:build`. Linux/Windows технически должно работать, но я гонял только на маке — если попробуешь, скажи как.

## Как пользоваться

1. Создай проект, укажи папку где будет жить код. В **«Продвинутых настройках»** выбери CLI (Claude Code или Codex) и модель — потом можно поменять через шестерёнку.
   Текст из поля **«Видение продукта»** сразу попадёт в `docs/product/vision.md`: это живой источник истины для PO и Architect.
2. Опиши видение проекта — одной строкой или абзацем.
3. (Опционально) **Кидай айтемы в беклог** ещё до старта: «нужен поиск», «использовать TypeScript», «не CRA». Каждому ставится категория (фича / идея / баг / крит проблема / техдолг). PO учтёт их в первой же итерации.
4. Жми Start. Можно вообще закрыть окно — приложение продолжит крутиться. Или смотреть в реальном времени: что делает каждый агент, граф задач, активность.
5. Когда захочешь посмотреть результат — кнопка **«Покажи, что есть»**. Дождись, пока поднимется preview.
6. Тыкаешь приложение, пишешь правки в беклог с категориями — можно несколько штук подряд. Жмёшь **«Продолжаем»**.

«Покажи, что есть» работает и сразу после создания проекта — если код ещё не пишется или работа на ранней стадии, Автономыч покажет состояние «как есть» и ждёт твоего пинка.

Если проект по дороге меняет направление, править нужно не старое поле «идея» в UI, а актуальное `docs/product/vision.md` или беклог. Поле в сайдбаре теперь просто пользовательская заметка для отображения.

## Беклог и категории — как работают правки

Все твои правки, идеи и баг-репорты идут в **беклог проекта** с одной из 5 категорий. Отдельного steering-инжекта в следующий PO-промпт больше нет: пользовательский input идёт только через backlog и `docs/product/vision.md`.

- 🚨 **Crit** — блокер. Проект не запускается, билд падает, демо мертво
- 🐛 **Bug** — известный баг, сломанный сценарий
- 🔧 **TechDebt** — риск, недоделка, рефакторинг
- ✨ **Feature** — новый функционал
- 💭 **Wish** — пожелание / nice-to-have

**PO в начале каждой итерации читает беклог** и сам выбирает 1-3 айтема по правилу: сначала закрыть всё `critical`, потом `bug`, потом `tech_debt`, потом `feature`/`wish`. Это не жёсткий инжект «сделай это сейчас» в промпт — PO видит структурированный список с категориями и решает сам по правилам в своём системном промпте. Можешь нарушить иерархию (например, баг требует фундаментальной переделки и лучше сначала закрыть смежный айтем), но **не пропустит баги ради новой фичи**.

Беклог наполняется не только тобой:
- **Reviewer** при ревью находит риски и недоделки → добавляет с категорией `bug`/`tech_debt`/`critical`
- **Упавшие таски** автоматически попадают в беклог с категорией `bug` и высоким приоритетом — следующая итерация их подберёт
- **PO** может парковать свои идеи в `add_to_backlog` — что не вошло в текущую итерацию

Беклог-панель в правой сайдбаре показывает всё это с фильтрами «Активные» / «Архив» и кнопками dismiss/edit.

## Что делать когда что-то идёт не так

- **Лимит токенов / 5-часовое окно у Claude.** Автономыч сам распознаёт rate-limit в ошибке агента, переходит в `Paused`, рисует countdown «продолжим через MM:SS» и **сам резюмит итерацию когда лимит восстановится**. Можешь нажать «Продолжить» чтобы попробовать раньше, или Stop чтобы остаться на паузе.
- **Хочешь сменить модель на ходу.** Жмёшь Stop → итерация переходит в `Paused` (не теряется!). Шестерёнка → меняешь модель / CLI / permission mode. «Продолжить итерацию N» — та же итерация продолжает с того места.
- **Ошибка агента (модель сошла с ума, что-то поломалось).** Автономыч пытается **один раз** довести итерацию до конца, при провале — пауза, чтобы ты разобрался. Никаких бесконечных retry-циклов которые жгут токены.
- **Презентер во время демо.** Под превью есть чат с Presenter-агентом — он может сам починить проблемы запуска (порты, env, зависимости) либо предложить добавить в беклог как баг.
- **Полный краш приложения.** Запусти заново — найдёт незавершённую итерацию, прибьёт сиротские worktree, продолжит с того места.

## Что внутри

- Нативное Tauri-приложение на Rust. Бандл ~10-15 МБ в релизе, RAM на старте ~80 МБ.
- **Два бэкенда агентов:**
  - `claude` CLI в режиме `stream-json` — стандартный для Claude Code
  - `codex` CLI в режиме `exec --json` — для OpenAI Codex (gpt-5.4 / gpt-5.3-codex / gpt-5.5)
  - Выбирается per-project. Контракт `AgentEvent` одинаковый — кондуктор не знает что под ним
- Каждая задача специалиста — отдельный `git worktree` на своей ветке. Параллельная работа без конфликтов, мерджи через rebase-onto-main + `--ff-only`, перед каждой итерацией ставится тег `autonomych/pre-iter-N` чтобы откатиться.
- **MergeResolver** агент: если rebase упирается в конфликт — запускается специальный агент в paused-rebase состоянии и разруливает контентно.
- **Documenter** агент: после ревью обновляет распределённую документацию проекта в `docs/` через INDEX-навигацию. При создании проекта Автономыч сам создаёт начальный `docs/product/vision.md` из пользовательского описания, а дальше Documenter держит его в актуальном состоянии. PO и Architect читают именно docs, а не зашитую строку `project.idea`.
- **Crash recovery + Pause/Resume:** убил приложение посреди итерации или нажал Stop — состояние сохраняется в JSON, при следующем Start подхватывается ровно с того места. Сиротские worktree и preview-процессы убираются.
- **Smart cooldown:** ловится rate-limit в ошибках агента, парсится время восстановления из `try again in N seconds` / Claude unix-timestamp / прочих форматов, автоматический resume по таймеру. Cap 6 часов, дефолт 30 минут если не удалось распарсить.
- **Blocker Reviewer:** если специалист реально упёрся — он пишет вопрос в специальном формате через `ask_user`. Перехватывает Blocker Reviewer: либо отвечает сам (если очевидно), либо прокидывает тебе в UI и ждёт.
- **Overseer chat:** сбоку — чат с агентом который видит всё состояние проекта и может объяснить что происходит, подсказать формулировку правки.
- **Loop detection:** если PO 3 раза подряд берёт одну и ту же тему — в его промпте появляется предупреждение «возможная петля, смени область или передай вопрос пользователю».

## Честные ограничения

- **Стабильность.** Это pet-проект, не продакшен. UI работает, цикл крутится, но e2e на больших проектах я не гонял — если найдёшь, где ломается, открой issue.
- **Стрим-форматы CLI — не публичные API.** Парсеры `claude` (`stream-json`) и `codex` (`exec --json`) могут поплыть при обновлении CLI — фиксится в `src-tauri/src/agents/claude.rs` / `codex.rs`. Неизвестные события логируются на WARN с raw-данными чтобы было видно что сломалось.
- **Стоит денег.** Агенты крутятся в фоне и жгут токены. Wrap-up по кнопке существует не просто так — оставленный на ночь Автономыч может уйти в неплохой счёт. Smart cooldown спасает от лимитов, но не от стоимости.
- **Permission mode по умолчанию — `bypassPermissions`.** Это нужно чтобы агенты могли запускать `Bash` без подтверждений. Запускай его на проектах, где не страшно — в отдельной папке, не в `$HOME`.
- **macOS only verified.** Preview-агент использует `setsid` для запуска dev-серверов — на Windows понадобится альтернатива.
- **Codex CLI и не-ASCII пути.** Codex CLI плохо работает с путями содержащими кириллицу/юникод — используй латинские имена папок для Codex-проектов. Claude такого ограничения нет.
- **Авто-failover между провайдерами пока нет.** Если у тебя один CLI лимитнулся — Автономыч ждёт его восстановления, не пытается переключить на другой. Это в планах.
